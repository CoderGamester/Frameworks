## 2026-04-26 — Confirmed AssetResolver sample-removal "user's undo" design (revert of self-cleanup attempt)

### Architectural Decisions
- Confirmed the original "removing the dedicated Addressables group is the user's undo" design for the AssetResolver sample after a self-cleanup attempt was found to be structurally impossible. The natural-feeling fix — `AssetPostprocessor.OnPostprocessAllAssets` watching `deletedAssets` for the post-processor's OWN script path and running cleanup synchronously — does not work: when a `.cs` is deleted, Unity recompiles BEFORE firing the deletion-batch callback, so the post-processor class no longer exists in the recompiled assemblies by the time the callback would fire. The two viable alternatives if cleanup were genuinely required are (a) move the cleanup into the package's main `Editor/` assembly with hardcoded constants for the sample's group/label names + an `[InitializeOnLoadMethod]` orphan-detector running on every domain reload in every consumer's project, or (b) accept the manual "user's undo" — for this sample we chose (b) because (a) violates the sample-scoped boundary and runs forever in every consumer's project for a feature only meaningful right after sample deletion.

### Structural Changes
