# Architecture Log

## 2026-04-26 — Services Explorer hardening + CoroutineService runtime contract fix

### Architectural Decisions
- Fixed two compounding bugs in `CoroutineService.AsyncCoroutine` introduced in v2.0.0 by the editor-tracking work: (a) `OnComplete(Action)` was replace-semantics, so installing the editor's `_activeAsyncCoroutines.Remove(...)` removal lambda silently overwrote (or was overwritten by) user callbacks for `StartDelayCall` / `StartAsyncCoroutine`; (b) `StopCoroutine(bool triggerOnComplete)` ignored its parameter and never flipped `IsCompleted` / `Coroutine`. Solution: introduced a private `event Action InternalCleanup` on `AsyncCoroutine` separate from the user-facing `_onComplete` field — editor tracking subscribes to `InternalCleanup`, never to `OnComplete`. `StopCoroutine` now flips state correctly, honors its `triggerOnComplete` flag, and is a no-op if already completed. Public `OnComplete` REPLACE semantics preserved for backwards compatibility.
- Established `ServiceTab.OnExitingPlayMode()` virtual + a `!EditorApplication.isPlaying` short-circuit at the top of every populated-state tab's `Refresh()` as the canonical pattern for decoupling Editor Explorer UI from `MainInstaller` cleanup correctness. The override clears the UI synchronously the moment Stop is pressed (before scene teardown); the short-circuit guards every subsequent refresh against any leftover static service state. Seven populated-state tabs adopted the pattern (Installer, MessageBroker, Data, RNG, Pool, Coroutine, Tick). The contextual banner latch (`_hasSeenPlay`) flips text from "Not in Play mode — showing last snapshot" → "Play session ended — services unbound".
- Standardized destructive-action styling via a new `.action-primary-danger` USS class + `MakePrimaryDangerButton` helper. Rewrote `.row-btn-danger` from text-color-only to tinted-bg + border + bold light-pink (matches `.status-err`'s contrast pattern); the previous styling failed contrast in both Personal and Professional editor skins.
- MessageBroker tab redesign: replaced the freeform "Type FQN" `TextField` with a reflection-populated `DropdownField` (one-time scan over `AppDomain.CurrentDomain` for non-abstract `IMessage` implementers, refreshed on assembly-count fingerprint change to avoid per-tick allocation), split "Test Publish" from the action bar into its own labeled section, and moved the per-type `Unsubscribe All` button onto the Foldout's `Toggle` header row (with `evt.StopPropagation()` on the button's `ClickEvent` so it doesn't also collapse the foldout).
- RngTab UX rebuild: always-on `Next value` row showing `IRngService.Peek` so the consume-vs-non-consume distinction is concrete; renamed "Restore" to "Set counter" + added `Rewind to 0` shortcut; replaced cramped `IntegerField` widgets with `Slider/SliderInt` + naked `IntegerField` pairs (matches `TimeTab.AddTime` bidirectional pattern with `RegisterValueChangedCallback` + `SetValueWithoutNotify`).
- Removed the Services Playground sample's "Reset spawn counts" button entirely after iteration concluded it added no `IPoolService` demonstration value (it only reset two sample-internal `Bullet.TotalSpawns` / `Bullet.TotalDespawns` static counters). The lifetime tally remains in the live-status panel for visibility into pool lifecycle hooks; it just has no reset affordance. Cleanup spanned six layers: C# field + wire-up + handler + the `Bullet.ResetCounters()` static helper, the prefab's `[SerializeField]` reference, the `Btn_Resetcounters` `GameObject` + 5 components, the orphaned child `Label` `GameObject` + 3 components, the parent layout group's `m_Children` entry, and 6 stale `PrefabModification` overrides in the host scene file.

### Data Flows
- AsyncCoroutine completion (natural): `IEnumerator` finishes → `Completed()` → `IsCompleted=true, Coroutine=null` → `OnCompleteTrigger()` (fires user `_onComplete`) → `InternalCleanup?.Invoke()` (fires editor's tracking removal lambda → entry leaves `_activeAsyncCoroutines`).
- AsyncCoroutine completion (explicit stop): `IAsyncCoroutine.StopCoroutine(triggerOnComplete)` → no-op if already completed → otherwise stop the underlying Unity coroutine, set `IsCompleted=true, Coroutine=null`, fire `OnCompleteTrigger()` iff `triggerOnComplete=true`, fire `InternalCleanup?.Invoke()` always. Same teardown shape regardless of natural-vs-explicit path.
- Explorer tab cleanup on Stop: `EditorApplication.playModeStateChanged(ExitingPlayMode)` → `ServiceTab.OnPlayModeChanged` → (1) `StopRefreshTimer()` → (2) `OnExitingPlayMode()` synchronously clears populated UI → (3) `UpdateBannerVisibility()` → (4) `EditorApplication.delayCall += DelayedExitRefresh` → after `MonoBehaviour.OnDestroy` runs in the bootstrap (which calls `MainInstaller.Clean()`) → `DelayedExitRefresh()` → `Refresh()` short-circuits on `!isPlaying` so it doesn't re-render against any leftover static state.
- MessageBrokerTab type-discovery: `RebuildMessageTypeChoicesIfStale()` compares the current `AppDomain.GetAssemblies().Length` to the last-scanned count; on change → full scan via `asm.GetTypes()` filter for non-abstract non-interface types where `typeof(IMessage).IsAssignableFrom(t)` → sort by `FullName` → label-collision pass uses `Type.Name` by default but falls back to `FullName` for ambiguous names → publish path resolves `_messageTypes[index]` and invokes `IMessageBrokerService.Publish<T>` via reflection with `Activator.CreateInstance(type)`.

## 2026-04-26 — Services AssetResolver sample auto-Addressables setup

### Architectural Decisions
- Added sample-scoped editor automation for the `AssetResolver` sample so first-import is zero-click. New file `Samples~/AssetResolver/Editor/AssetResolverSampleSetup.cs` (assembly `GameLovers.Services.Samples.AssetResolver.Editor`, namespace `GameLovers.Services.Samples.AssetResolver.Editor`) lives inside the sample folder so removing the sample also removes the automation — no orphan `[InitializeOnLoad]` handler running in consumer projects after sample deletion.
- Triple-trigger redundancy: `AssetPostprocessor.OnPostprocessAllAssets` (sprite drops), `[InitializeOnLoadMethod]` safety net (UPM first-import chicken-and-egg, since the post-processor inside `Samples~/.../Editor/` compiles AFTER the import batch that brings it into the user's project and would otherwise miss its own first invocation), plus a `Tools > GameLovers > Samples > Asset Resolver > Refresh Addressables` menu item and a sample-scoped inspector button on `AssetConfigsScriptableObjectEditor`.
- Cross-editor-assembly decoupling: the package's main `Editor/` assembly does NOT reference the sample editor assembly. The inspector button invokes the sample's menu via `EditorApplication.ExecuteMenuItem("Tools/GameLovers/Samples/Asset Resolver/Refresh Addressables")`; visibility is guarded by an asset-path check (only shown when `target` ends with `/Asset Resolver/SpriteConfigs.asset`).
- Idempotent state-based design: setup is safe to re-run, suppresses logs in the silent path when no changes occurred, and never overwrites user mappings to other sprites (skips a row when its `m_AssetGUID` is non-empty and differs from the canonical sprite's GUID). Uses a dedicated Addressables group `GameLoversServicesSamples_AssetResolver` to avoid polluting user-defined groups; group deletion is the user's "undo".
- Three-pass rename strategy for non-canonical sprite filenames: exact match → substring match (e.g. `MyHeroIcon.png` → `Hero.png`) → alphabetical fallback. Strict matching against the `SpriteId` enum values (`Hero`/`Coin`/`Enemy`).

### Structural Changes
- `Packages/com.gamelovers.services/Samples~/AssetResolver/Editor/` (new): `AssetResolverSampleSetup.cs` + deterministic `.cs.meta` (`...d605`); `GameLovers.Services.Samples.AssetResolver.Editor.asmdef` + `.meta` (`...d606`) referencing `GameLovers.Services` + Addressables Runtime/Editor + ResourceManager via stable asmdef GUIDs.
- `Packages/com.gamelovers.services/Samples~/AssetResolver/Sprites/` (new in source, was working-tree-only): three placeholder PNGs renamed from descriptive ChatGPT-generated filenames to canonical `Hero.png`/`Coin.png`/`Enemy.png` so first import is a clean no-op for the rename pipeline.
- `Editor/Inspectors/AssetConfigsScriptableObjectEditor.cs`: added sample-scoped "Refresh AssetResolver Sample Addressables" button (path-guarded), `using System;`, and indentation fix on the previously misaligned `OnRegenerateIds`.
- Docs: `Samples~/AssetResolver/README.md` rewritten (auto-flow + swap-in-your-own-sprites + manual fallback); `Samples~/README.md` sample-2 row + file list updated; `package.json` `samples[]` description for "Asset Resolver" rewritten; `CHANGELOG.md` 2.0.0 entry expanded; `AGENTS.md` §3 sample row + §4 *Sample Zero-Setup Invariant* gained a paragraph spelling out the sample-scoped/cross-assembly/idempotency contract.

### Data Flows
- UPM first import: package files copied → editor scripts compile → `[InitializeOnLoadMethod]` fires → `EditorApplication.delayCall` → `RunSilent` → `AddressableAssetSettingsDefaultObject.GetSettings(create:true)` → `settings.FindGroup(GroupName) ??
