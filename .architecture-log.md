## 2026-04-26 — Confirmed AssetResolver sample-removal "user's undo" design (revert of self-cleanup attempt)

### Architectural Decisions
- Confirmed the original "removing the dedicated Addressables group is the user's undo" design for the AssetResolver sample after a self-cleanup attempt was found to be structurally impossible. The natural-feeling fix — `AssetPostprocessor.OnPostprocessAllAssets` watching `deletedAssets` for the post-processor's OWN script path and running cleanup synchronously — does not work: when a `.cs` is deleted, Unity recompiles BEFORE firing the deletion-batch callback, so the post-processor class no longer exists in the recompiled assemblies by the time the callback would fire. The two viable alternatives if cleanup were genuinely required are (a) move the cleanup into the package's main `Editor/` assembly with hardcoded constants for the sample's group/label names + an `[InitializeOnLoadMethod]` orphan-detector running on every domain reload in every consumer's project, or (b) accept the manual "user's undo" — for this sample we chose (b) because (a) violates the sample-scoped boundary and runs forever in every consumer's project for a feature only meaningful right after sample deletion.

### Structural Changes
- `Samples~/AssetResolver/Editor/AssetResolverSampleSetup.cs`: post-processor restored to single-branch import-only with an explicit XML-doc disclaimer (`This post-processor does NOT and CANNOT detect sample removal via deletedAssets`) so a future agent doesn't re-attempt the dead pattern.
- `Packages/com.gamelovers.services/AGENTS.md` §3 row + §4: gotcha "Sample-removal cannot self-cleanup" replaces the (now-deleted) "Sample-removal cleanup detection" instruction; documents the recompile-before-postprocess ordering, lists both viable paths, names the chosen one. Per-sample `README.md` now explains the manual cleanup steps as the user's undo.

## 2026-04-26 — Addressable Ids tab freshness banner + on-demand stale-Ids diff

### Architectural Decisions
- Established a two-tier observability model for editor tabs whose underlying scan is non-trivially expensive: **cheap path runs on every `Refresh()` tick** (file-stat-only via `File.GetLastWriteTime` against `AddressableAssetSettings.asset` plus each non-readonly group asset, ~20 stats max), **expensive path runs only on user button click** (`GetAssetList` + `ProcessData` once). Encoded as `AddressableIdsGeneratorUtils.ComputeFreshness` (cheap) + `Diff` (expensive) with a `FreshnessResult`/`DiffResult` mirror of the existing `GenerationResult` pattern. Rejected the obvious "live-diff every tick" option and the `AddressableAssetSettings.OnModification` event-subscription option — both add complexity disproportionate to value when an mtime-vs-mtime check answers "is the file stale?" in microseconds.
- Persisted last-generation snapshot in the existing `AddressableIdsEditorSettings` `ScriptableSingleton` rather than a sidecar file (`Library/AddressableIdsSnapshot.json`). Reasons: project-shared via VCS so all contributors see the same diff baseline; no separate lifecycle file to maintain; survives domain reload; atomic with the settings the snapshot was generated against (`_lastGenerationFilenameUsed` / `_lastGenerationLabelFilterUsed` ride along so the diff can flag "settings changed since last gen"). `RecordGeneration(addresses, labels)` is the only write path, called once at the end of `AddressableIdsGeneratorUtils.Generate` after `AssetDatabase.Refresh()`.
- Snapshot stored as ordinally pre-sorted `string[]` enables O(n+m) sorted set-diff on the read path (`SortedSetDiff` private helper) — no per-tick re-sorting, no `HashSet` allocation per diff. Sort happens once at write time.
- Preserved a pre-existing quirk in `GenerateAddressEnums` rather than fixing it as part of the refactor: the `name_filetype` collision-fallback updates the `addedNames` tracking list but the StringBuilder still emits the un-suffixed `GetCleanName(...)` for every entry. This means duplicate sanitized names produce CS0102 in the generated enum at consumer compile time. Extracted `ResolveSanitizedEnumName` as a shared helper used by both `GenerateAddressEnums` (preserves quirk) and the new `DetectSanitizedNameCollisions` (used for warnings) so a future fix can be made in one place — but flagged it explicitly as a "do not silently change" item in `AGENTS.md` §4 and in this log.

### Structural Changes
- `Packages/com.gamelovers.services/Editor/AddressableIds/AddressableIdsEditorSettings.cs`: 7 new `[SerializeField]` snapshot fields (`_lastGenerationUtcTicks`, id/label counts, filename + label-filter used, sorted `string[]` addresses + labels), `internal void RecordGeneration(IReadOnlyList<string>, IReadOnlyList<string>)` helper that ordinally sorts both inputs and calls `Save(true)`, and read-only public accessors (`HasSnapshot`, `LastGenerationUtc`, `LastGenerationIdCount`, `LastGenerationLabelCount`, `LastGenerationFilenameUsed`, `LastGenerationLabelFilterUsed`, `LastGenerationAddresses`, `LastGenerationLabels`).
- `Packages/com.gamelovers.services/Editor/AddressableIds/AddressableIdsGeneratorUtils.cs`: new `DiffResult` and `FreshnessResult` `internal readonly struct` mirrors of `GenerationResult`; new `Diff(settings)` runs the same `GetAssetList` + `ProcessData` pipeline as `Generate` but writes nothing and skips `AssetDatabase.Refresh`; new `ComputeFreshness(settings)` performs file-stat-only mtime comparison; new `ResolveScriptPath(settings)` extracted as a shared helper for tab + freshness paths; new private `ResolveSanitizedEnumName`, `DetectSanitizedNameCollisions`, `DetectNullAssetTypes`, `SortedSetDiff`, `ExtractAddresses` helpers. `Generate` now ends with a single `settings.RecordGeneration(...)` call.
- `Packages/com.gamelovers.services/Editor/Explorer/Tabs/AddressableIdsTab.cs`: full rewrite. Output section replaced — `_freshnessBanner` (color-coded: green = fresh, amber = stale, red = not generated yet), `_outputPathLabel`, persisted `_lastGenerationLabel` ("Last generated: 5m ago — N ids, M labels (filename: X, filter: Y)"), `Reveal file` + `Open Settings file` buttons. New "Pending changes" section between Output and the bottom action bar — `Check for stale Ids` button + summary line + collapsed `Foldout` with three sub-`VisualElement`s (added / removed / warnings); `OnCheckStale` is the only call site for `Diff`; button is disabled in `try/finally` during the scan; `OnGenerate` resets the pending section because the snapshot just changed.
- `Packages/com.gamelovers.services/AGENTS.md`: §3 `Editor/AddressableIds/` row expanded to mention the persisted last-generation snapshot, the new `Diff`/`ComputeFreshness` helpers, and the cost-tier separation. §4 gained two bullets: "snapshot is project-shared via VCS — do not add to `.gitignore`" and "`AddressableIdsTab` cost separation is on-demand-only by intent — do not move `Diff` into `Refresh()` to make it 'live'".
- `Packages/com.gamelovers.services/CHANGELOG.md`: two bullets folded into the existing unreleased `## [2.0.0]` `**New**:` block per the unreleased-version policy (no version bump, no `MIGRATION.md` entry).

### Data Flows
- Tab refresh (per tick, ~250 ms in Play mode, single-shot in Edit mode): `AddressableIdsTab.Refresh` → `RefreshOutput` → `AddressableIdsGeneratorUtils.ComputeFreshness(settings)` → file-stat `AddressableAssetSettings.asset` + each non-readonly group `.asset` → `max(latestSourceWriteTime)` vs `File.GetLastWriteTime(generatedScript)` → freshness banner color + text. Snapshot accessors read from in-memory `ScriptableSingleton` state. **No `GetAssetList` / `ProcessData` ever runs on this path.**
- Stale diff (button click only): `AddressableIdsTab.OnCheckStale` → `AddressableIdsGeneratorUtils.Diff(settings)` → `GetAssetList` + `ProcessData` (proportional to total Addressable entries) → ordinal sort current addresses + labels → `SortedSetDiff` against `settings.LastGenerationAddresses` / `LastGenerationLabels` → collision + null-AssetType detection → `DiffResult` rendered into the foldout (added / removed lists + warnings).
- Snapshot write (one path only): `AddressableIdsGeneratorUtils.Generate` → after `AssetDatabase.Refresh()` → `settings.RecordGeneration(addresses, labels)` → ordinal-sort both inputs → write `[SerializeField]` fields → `Save(true)` → `ProjectSettings/AddressableIdsEditorSettings.asset`.

## 2026-04-26 — AssetResolver sample full Explorer coverage + Foldout/refresh hardening

### Architectural Decisions
- Closed the gap where the `AssetResolver` sample only exercised one of its three target Services Explorer tabs. Three layered changes turn the sample into a genuine demo of all three tabs the `ServicesPlayground` doesn't cover: (1) the sample driver now binds `IAssetResolverService` through `MainInstaller` (mirrors `ServicesBootstrap`), so the **Asset Resolver** tab populates from the live `AssetMap` while Play is running; (2) a new sample-scoped concrete importer `SpriteConfigsImporter : AssetsConfigsImporter<SpriteId, Sprite, SpriteConfigs>` ships in the sample editor assembly, so `AssetsImporterEditorUtils.DiscoverImporters` reflection-scan surfaces a sample row in the **Assets Importer** tab; (3) the sample's Addressables post-processor now applies a single shared label `services-sample-asset-resolver` to its sprites so the **Addressable Ids** tab can demo a sample-scoped `Generate` against a label filter. Project-wide `AddressableIdsEditorSettings` is deliberately not auto-mutated — the README walks the user through the three field values so the demo teaches the tab UX rather than mutating user-owned `ScriptableSingleton` state.
- Established that any UPM sample whose editor code uses sample runtime types in generic constraints MUST ship a dedicated runtime `*.asmdef`. Unity disallows asmdef-defined assemblies from referencing `Assembly-CSharp`, so the editor asmdef cannot see `SpriteId` / `SpriteConfigs` until the sample runtime is its own assembly. New `Samples~/AssetResolver/GameLovers.Services.Samples.AssetResolver.asmdef` references `GameLovers.Services`, `GameLovers.GameData` (transitive — `Pair<TKey,TValue>` returned by `AssetConfigsScriptableObject.Configs`), and the engine packages the sample runtime touches; the editor asmdef references the new runtime asmdef so generic-constrained editor types compile cleanly.
- Sample-scoped Explorer-jump pattern: the sample editor assembly registers a new menu `Tools > GameLovers > Samples > Asset Resolver > Open in Explorer` that calls `ServicesExplorerWindow.OpenOnTab<AssetResolverTab>()`. The sample's runtime UI's "Open Services Explorer" button invokes the menu via `EditorApplication.ExecuteMenuItem` under `#if UNITY_EDITOR`, so the runtime sample assembly never has to reference `GameLovers.Services.Editor` (matches the `AssetConfigsScriptableObjectEditor` sample-scoped refresh button pattern from the prior session).
- Hardened the Services Explorer's periodic-rebuild refresh model to survive interactive content. Three layered fixes on `ServiceTab` and the two rebuild-style tabs (`AssetResolverTab`, `MessageBrokerTab`): (a) `MakeStickyFoldout(key, text, defaultExpanded)` persists per-tab user-collapsed state in a `HashSet<string>` keyed by stable identity (e.g., `assetType.FullName + "/" + idType.FullName`), so `_tree.Clear()`-style refreshes don't reset to `Foldout` default `value = true` every 250 ms; (b) `MakeStickyFoldout`'s value-changed callback filters with `if (evt.target != foldout) return;` because UIToolkit `ChangeEvent<bool>` bubbles up the visual tree — without the filter, collapsing an inner foldout silently marks the outer foldout collapsed too, making the Asset Resolver tab's `AssetType → IdType` two-level tree collapse everything when only the inner level was clicked; (c) `TryShortCircuitRefresh(string digest)` lets a tab return early from `Refresh()` when the displayed data is unchanged, which is what eliminates rapid-click lost input — without it, the periodic timer destroys mouse-captured `VisualElement`s mid-click and Unity loses the mouse-up against a deleted target. Each tab computes a deterministic digest from every piece of state the rebuild path conditions on (per-row identity + load status + render-gating UI flags); the destructive-toggle on `AssetResolverTab` wires `RegisterValueChangedCallback(_ => InvalidateRefreshDigest())` so flipping it forces a rebuild even when data is unchanged.

## 2026-04-26 — Services Explorer hardening + CoroutineService runtime contract fix

### Architectural Decisions
- Fixed two compounding bugs in `CoroutineService.AsyncCoroutine` introduced in v2.0.0 by the editor-tracking work: (a) `OnComplete(Action)` was replace-semantics, so installing the editor's `_activeAsyncCoroutines.Remove(...)` removal lambda silently overwrote (or was overwritten by) user callbacks for `StartDelayCall` / `StartAsyncCoroutine`; (b) `StopCoroutine(bool triggerOnComplete)` ignored its parameter and never flipped `IsCompleted` / `Coroutine`. Solution: introduced a private `event Action InternalCleanup` on `AsyncCoroutine` separate from the user-facing `_onComplete` field — editor tracking subscribes to `InternalCleanup`, never to `OnComplete`. `StopCoroutine` now flips state correctly, honors its `triggerOnComplete` flag, and is a no-op if already completed. Public `OnComplete` REPLACE semantics preserved for backwards compatibility.
- Established `ServiceTab.OnExitingPlayMode()` virtual + a `!EditorApplication.isPlaying` short-circuit at the top of every populated-state tab's `Refresh()` as the canonical pattern for decoupling Editor Explorer UI from `MainInstaller` cleanup correctness. The override clears the UI synchronously the moment Stop is pressed (before scene teardown); the short-circuit guards every subsequent refresh against any leftover static service state. Seven populated-state tabs adopted the pattern (Installer, MessageBroker, Data, RNG, Pool, Coroutine, Tick). The contextual banner latch (`_hasSeenPlay`) flips text from "Not in Play mode — showing last snapshot" → "Play session ended — services unbound".
- Standardized destructive-action styling via a new `.action-primary-danger` USS class + `MakePrimaryDangerButton` helper. Rewrote `.row-btn-danger` from text-color-only to tinted-bg + border + bold light-pink (matches `.status-err`'s contrast pattern); the previous styling failed contrast in both Personal and Professional editor skins.
- MessageBroker tab redesign: replaced the freeform "Type FQN" `TextField` with a reflection-populated `DropdownField` (one-time scan over `AppDomain.CurrentDomain` for non-abstract `IMessage` implementers, refreshed on assembly-count fingerprint change to avoid per-tick allocation), split "Test Publish" from the action bar into its own labeled section, and moved the per-type `Unsubscribe All` button onto the Foldout's `Toggle` header row (with `evt.StopPropagation()` on the button's `ClickEvent` so it doesn't also collapse the foldout).

The lifetime tally remains in the live-status panel for visibility into pool lifecycle hooks; it just has no reset affordance. Cleanup spanned six layers: C# field + wire-up + handler + the `Bullet.ResetCounters()` static helper, the prefab's `[SerializeField]` reference, the `Btn_Resetcounters` `GameObject` + 5 components, the orphaned child `Label` `GameObject` + 3 components, the parent layout group's `m_Children` entry, and 6 stale `PrefabModification` overrides in the host scene file.

- Introduced a shared runtime-compatible test-helpers assembly (`Tests/Helpers/` asmdef gated by `UNITY_INCLUDE_TESTS`) in `com.gamelovers.uiservice`, enabling EditMode and PlayMode tests to share MonoBehaviour-derived fixtures that `AddComponent<T>()` can legally attach to GameObjects.
- Preferred real pool fixtures over NSubstitute mocks for tests targeting self-referentially-constrained generic interfaces (e.g., `IObjectPool<T>` where `T : IPoolEntityObject<T>`) to work around a Castle DynamicProxy `ArgumentNullException: localType` IL-emission bug on Unity's Mono runtime.

### Structural Changes
- `com.gamelovers.services/Runtime/Pooling/ObjectPool.cs`: fixed iterate-while-mutating bug in `Despawn(bool onlyFirst, Func<T,bool>)` — forward index was not compensated after `SpawnedEntities.Remove`, so adjacent matches were skipped.
- `com.gamelovers.services/Tests/EditMode/Unit/ObjectPoolTest.cs`: rewrote `EntityDespawn_Successfully` to use a real `ObjectPool<IMockEntity>` instead of `Substitute.For<IObjectPool<IMockEntity>>()`.
- `com.gamelovers.services/Tests/EditMode/Unit/AssetResolverServiceTest.cs`: qualified `Object.DestroyImmediate(so)` as `UnityEngine.Object.DestroyImmediate(so)` to resolve CS0104 ambiguity introduced by `using System;` + `using UnityEngine;` in the same file.
- `com.gamelovers.gamedata/Tests/Editor/Unit/TestDataBuilders.cs` + `ConfigBrowser/ConfigValidationServiceTests.cs`: added `MockValidatableConfigAlt` sibling type; test now registers singleton and collection under two different types because `ConfigsProvider` keys its storage by `Type` and each type can hold only one role.
- `com.gamelovers.uiservice/Tests/Helpers/` (new): runtime-compatible helpers asmdef with `TestPresenter.cs` moved out of `Tests/EditMode/Helpers/`; both EditMode and PlayMode test asmdefs now reference it.
