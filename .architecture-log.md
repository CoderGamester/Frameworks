# Architecture Log

## 2026-04-24 — Services Explorer tool migration + Overview tab + menu consolidation

### Architectural Decisions
- Consolidated the legacy `[CustomEditor]`-hosted tools (`AssetsImporter.asset`, `AddressablesIdGeneratorSettings.asset`) into first-class Services Explorer tabs; hard-deleted the ScriptableObject wrappers (no `[Obsolete]` shim) and migrated their state to `ScriptableSingleton` in `ProjectSettings/` — mirrors the `VersioningEditorSettings` precedent so every tool has identical settings-storage plumbing.
- Introduced a landing **`OverviewTab`** as the first tab — per-service card grid with bound/ready status pills + `Open` jump-link + each tab's primary CTA; chosen as the "single pane of glass" entry point so users aren't forced to remember which tab owns which bulk action.
- Established a **settings-vs-verbs menu policy**: `Tools > GameLovers > ...` holds only editor-time one-shot verbs; settings and toggles (e.g. `AutoUpdateOnRefresh`) live exclusively in tabs. `Toggle Auto Import On Refresh` deleted from the menu as a concrete instance.
- Chose **nested submenus** (`Tools/GameLovers/<Tool>/{<Primary Verb>, Open in Explorer}`) over flat + separator — each tool submenu has its verb at priority 100 and `Open in Explorer` at priority 200, which Unity auto-separates via the >=11 priority-gap rule.
- Normalized a single primary CTA per tab via `ServiceTab.MakePrimaryButton` + `.action-primary` USS class — retrofitted across all 10 existing tabs so the Overview card's primary button mirrors the tab's own action bar.
- Tab-jump API `ServicesExplorerWindow.SelectTab<T>()` / `OpenOnTab<T>()` constrained to `where T : ServiceTab` (tab types, not service interfaces) — the Overview cards pass tab-type arguments rather than `IInstaller`/`IMessageBrokerService`/etc.
- Pure-logic utility classes (`AssetsImporterEditorUtils`, `AddressableIdsGeneratorUtils`) extracted from the deleted `[CustomEditor]` wrappers so generation/import logic is shared across tabs, menu stubs, and the `AssetConfigsScriptableObjectEditor` inspector button (which now calls the util directly, eliminating the `ExecuteMenuItem` round-trip).
- CHANGELOG policy: since `com.gamelovers.services` 2.0.0 is still unreleased, all entries land under a new `## [Unreleased]` section at the top of `CHANGELOG.md` with `package.json` version untouched.

### Structural Changes
- New folder `com.gamelovers.services/Editor/AddressableIds/` (ns `GameLovers.Services.AddressableIds.Editor`) with `AddressableIdsEditorSettings.cs` (ScriptableSingleton with `IsValidIdentifier`/`IsValidNamespace`), `AddressableIdsGeneratorUtils.cs` (returns `GenerationResult`), `AddressableIdsMenu.cs`.
- New files in `Editor/AssetsImporter/`: `AssetsImporterEditorSettings.cs`, `AssetsImporterEditorUtils.cs`, `AssetsImporterMenu.cs`.
- New file `Editor/Versioning/VersioningMenu.cs` (`Refresh Version Data`, `Open in Explorer` stubs).
- New tabs in `Editor/Explorer/Tabs/`: `OverviewTab.cs` (takes `ServicesExplorerWindow` ref), `AssetsImporterTab.cs`, `AddressableIdsTab.cs`; `ServicesExplorerWindow.cs` registers 13 tabs and exposes `SelectTab<T>() where T : ServiceTab` + `OpenOnTab<T>()`; `ServiceTab.cs` gained `MakePrimaryButton` helper.
- `Editor/Explorer/Windows/ServicesExplorerWindow.uss` gained `.action-primary`, `.overview-grid`, `.overview-card*` classes.
- All 10 existing tabs retrofitted to `MakePrimaryButton` for their primary action; `PoolTab`/`DataTab`/`TimeTab`/`AssetResolverTab` gained new bulk-action methods (`OnClearAllPools`, `OnSaveAll`, `OnResetTime`, `OnUnloadAll`).
- `Editor/Inspectors/AssetConfigsScriptableObjectEditor.cs` now calls `AddressableIdsGeneratorUtils.Generate(AddressableIdsEditorSettings.instance)` directly.
- Deleted legacy `Editor/AssetsImporter/{AssetsImporter, AssetsToolImporter, AddressableIdsGenerator, AddressablesIdGeneratorSettings}.cs` + `.meta` via `git -C Packages/com.gamelovers.services rm -f`.
- Docs: `AGENTS.md` §3 (Editor folder map rewritten), §4 (new gotcha paragraphs), §8 (new update-triggers); `CHANGELOG.md` `## [Unreleased]` section; `README.md` Services Explorer table expanded to 13 rows with a primary-CTA column.
- Tests: `Tests/EditMode/Unit/AddressableIdsEditorSettingsTest.cs` with 14 validator tests (empty, whitespace, leading digit, invalid chars, dot/consecutive-dot, valid defaults, deep hierarchy).

### Data Flows
- Menu → utility: `Tools > GameLovers > Assets Importer > Import Assets Data` → `AssetsImporterEditorUtils.ImportAll` → `AssetDatabase.SaveAssets + Refresh`. Parallel flows for `Addressable Ids > Generate` and `Versioning > Refresh Version Data`.
- Tab → settings: tab UI reads/writes via `<X>EditorSettings.instance`, setter persists to `ProjectSettings/<X>EditorSettings.asset` on every value change.
- Overview → tab: card `Open` button → `_window.SelectTab<TTab>()` → `TabView.activeTab` update → `ServiceTab.Refresh()` fires on attach.
- Inspector → utility: `AssetConfigsScriptableObjectEditor.OnRegenerateIds` → `AddressableIdsGeneratorUtils.Generate(AddressableIdsEditorSettings.instance)` (no `ExecuteMenuItem` round-trip).

## 2026-04-24 — AddressablesAssetLoader hardening + UnloadAsset API rename

### Architectural Decisions
- Removed the `GC.Collect()` + `GC.WaitForPendingFinalizers()` + `Resources.UnloadUnusedAssets()` teardown sequence from per-asset Addressables release; memory reclamation is now exclusively the caller's responsibility at scene transitions / boot / memory-pressure events. Encoded as a "do NOT add back" guard in the package `AGENTS.md` §4.
- Renamed `IAssetLoader.UnloadAssetAsync<T>(T, Action)` → `UnloadAsset<T>(T, Action)` and changed the return type from `UniTask` to `void` so the signature reflects the method's actually-synchronous behavior; accepted the one-line breaking change on downstream callers (documented in `MIGRATION.md` §9).
- Deleted `AddressablesAssetLoaderIntegrationTest` rather than maintaining it: the wrapper has no branching logic worth integration-testing, `[Explicit]` tests with live-Addressables setup requirements rot, and the stale placeholder key was already triggering editor crashes. Replaced the `Tests/AGENTS.md` §9 `[Explicit]` block with a "why this wrapper isn't integration-tested" rationale so future contributors don't reintroduce it.
- Used `InternalsVisibleTo` to expose the intentionally-`internal` `VersioningEditorSettings` to the editor test assembly instead of widening its visibility; mirrored the existing `Runtime/AssemblyInfo.cs` pattern with a new `Editor/AssemblyInfo.cs`.

### Structural Changes
- `com.gamelovers.services/Editor/AssemblyInfo.cs` (new): `[assembly: InternalsVisibleTo("GameLovers.Services.Editor.Tests")]` — unblocks `VersioningEditorSettingsTest` without changing runtime API surface.
- `com.gamelovers.services/Runtime/AssetsImporter/IAssetLoader.cs`: method renamed, return type changed, XML doc rewritten (removed the incorrect "will destroy GameObject instances" claim — `Addressables.Release(gameObject)` does not destroy; only `Addressables.ReleaseInstance` does).
- `com.gamelovers.services/Runtime/AssetsImporter/AddressablesAssetLoader.cs`: implementation body now `Addressables.Release(asset) + callback` only; all `System.GC` and `Resources.UnloadUnusedAssets()` calls removed.
- `com.gamelovers.services/Tests/PlayMode/Integration/AddressablesAssetLoaderIntegrationTest.cs` + `.meta` deleted.
- Documentation alignment: `AGENTS.md` §4, `docs/asset-loading.md`, `CHANGELOG.md` [2.0.0] (Fixed + Breaking Changes), `MIGRATION.md` §9 (new), `Tests/AGENTS.md` §9.

### Data Flows
- Per-asset unload path: consumer → `IAssetLoader.UnloadAsset(asset)` → `Addressables.Release(asset)` → synchronous callback invocation. Memory reclamation is decoupled and scheduled by the consumer via `Resources.UnloadUnusedAssets()` at scene transitions, or implicitly by Unity on `LoadSceneMode.Single` scene loads.

## 2026-04-24 — Versioning tab: configurable write path + tab reorder

### Architectural Decisions
- Chose editor-only `ScriptableSingleton<T>` persisted to `ProjectSettings/VersioningEditorSettings.asset` rather than `EditorPrefs` or a runtime `ScriptableObject` — keeps runtime code unchanged and provides a typed POCO accessible from both UI and non-UI editor code.
- Removed the Regenerate (dev)/(store) manual buttons because dev regeneration already fires automatically via `[InitializeOnLoadMethod]`; the store variant is intentionally kept only as a build-pipeline API (`VersionEditorUtils.SetAndSaveInternalVersion(true)`).
- Enforced that the configurable folder must contain a `Resources` segment (hard error, not warning) to guarantee `Resources.Load<TextAsset>("version-data")` continues to work regardless of where the file is written.

### Structural Changes
- `Editor/Versioning/VersioningEditorSettings.cs` (new): `ScriptableSingleton` holding the write folder path with a static `IsValidResourcesPath` validator.
- `Editor/Versioning/VersionEditorUtils.cs`: replaced hardcoded `FilePath = "Configs/Resources"` const with `VersioningEditorSettings.instance.ResourcesFolderPath`; derives absolute path via `Directory.GetParent(Application.dataPath)` so any project-relative `Assets/...` path works.
- `Editor/Explorer/Tabs/VersioningTab.cs`: added "Write Location" section (Browse + Reset buttons) above "Runtime Values"; removed Regenerate buttons and their handlers.
- `Editor/Explorer/Windows/ServicesExplorerWindow.cs`: Versioning tab moved to index 0 in `RegisterTabs()`.

### Data Flows
- Write path: `VersioningTab` browse action → `VersioningEditorSettings.SetResourcesFolderPath` → `ProjectSettings/VersioningEditorSettings.asset`; `VersionEditorUtils.SaveVersionData` reads the same settings on every domain-reload write.

## 2026-04-23 — Services Explorer window + editor introspection bridge

### Architectural Decisions
- Established an `InternalsVisibleTo("GameLovers.Services.Editor")` bridge as the sole mechanism for editor tooling to inspect runtime service state — public runtime API stays untouched and retail builds pay zero cost.
- Selected UIToolkit (UXML + USS + Unity 6 `TabView` + `CreateInspectorGUI` / `CreatePropertyGUI`) as the rendering stack for all new editor tooling in `com.gamelovers.services`; IMGUI is no longer the default for new work.
- Adopted an abstract `ServiceTab : VisualElement` base with play-mode-aware scheduling (`schedule.Execute(...).Every(250)` live refresh during Play, static snapshot with banner during Edit) as the canonical pattern for per-service editor panels.
- Editor-only tracking state (e.g. active `IAsyncCoroutine` list in `CoroutineService`) is gated behind `#if UNITY_EDITOR` to maintain the package's zero-per-frame-allocation promise in retail builds.
- Scaffolder UX mirrors Unity's built-in "Create > C# Script" flow via `ProjectWindowUtil.StartNameEditingIfProjectWindowExists` + custom `EndNameEditAction` + `.cs.txt` templates in `Templates~/` (tilde suffix excludes templates from Unity's asset import).
- Pre-publication versioning policy: folded all new editor-UX work into the still-unreleased `[2.0.0]` CHANGELOG section rather than bumping to `[2.1.0]`, since the package has not yet been published.

### Structural Changes
- `com.gamelovers.services/Runtime/AssemblyInfo.cs` (new): single `[assembly: InternalsVisibleTo("GameLovers.Services.Editor")]`.
- `com.gamelovers.services/Runtime/*.cs` (9 files): added `internal` read-only accessors on `Installer`, `MainInstaller`, `MessageBrokerService`, `TickService` (+ `TickData` promoted to internal struct), `CoroutineService` (+ `#if UNITY_EDITOR` tracking list), `PoolService`, `DataService`, `TimeService`, `AssetResolverService`.
- `com.gamelovers.services/Editor/Explorer/` (new): `Windows/ServicesExplorerWindow.{cs,uxml,uss}` + `Tabs/ServiceTab.cs` (abstract) + 10 per-service tab files (Installer, MessageBroker, Tick, Coroutine, Pool, Data, Time, Rng, AssetResolver, Versioning).
- `com.gamelovers.services/Editor/Inspectors/` (new): `AssetConfigsScriptableObjectEditor.cs` (diagnostics panel + default `InspectorElement` + Regenerate Ids button) and `AssetReferenceSceneDrawer.cs` (UIToolkit property drawer).
- `com.gamelovers.services/Editor/AssetsImporter/AddressableIdsGenerator.cs`: in-place upgrade of `AddressablesIdGeneratorSettingsEditor` from IMGUI (`OnInspectorGUI`) to UIToolkit (`CreateInspectorGUI`).
- `com.gamelovers.services/Editor/Scaffolders/` (new): `ServicesScaffolders.cs` + `Templates~/NewMessage.cs.txt`, `NewCommand.cs.txt`, `NewService.cs.txt`, `NewPoolEntity.cs.txt`.
- Package `AGENTS.md` §3 extended with new editor folder map; §4 gained "Editor Introspection (InternalsVisibleTo)" subsection with the accessor table; §8 update policy extended. README.md gained an "Editor Tools" section.

### Data Flows
- Editor Explorer reads runtime state path: `MainInstaller.InstallerInstance.Bindings` → `ITickService`/etc. resolved at the tab layer → `internal` accessor on concrete service → `IReadOnlyDictionary`/`IReadOnlyList` projection → UIToolkit `VisualElement` re-populated every 250 ms during Play mode.

## 2026-04-22 — Documentation convention: CLAUDE.md wrapper + README/AGENTS split

### Architectural Decisions
- Adopted a three-file documentation convention across all UPM packages that carry an `AGENTS.md`: `README.md` for end users, `AGENTS.md` as single source of truth for AI agents + human contributors, and `CLAUDE.md` as a thin wrapper that imports `AGENTS.md` via Claude Code's native `@AGENTS.md` syntax (no duplicated content).
- Formalized "Option B" README shape repo-wide: lean Why/Install/Quick Start/Services-at-a-Glance/Samples/Related-docs/Contributing/Support/License, with deep API reference moving to a sibling `docs/` folder only when the trimmed README would otherwise exceed ~350 lines.
- Host `AGENTS.md` §8 now documents the `CLAUDE.md` convention as a repo-wide rule so new packages get the same treatment automatically.

### Structural Changes
- Added `CLAUDE.md` + `CLAUDE.md.meta` at the root of four packages: `com.gamelovers.services`, `com.gamelovers.uiservice`, `com.gamelovers.gamedata`, `com.gamelovers.mobileservices` (each meta uses the `TextScriptImporter` template).

- Per-package README trims: services 701→279 lines, gamedata 559→209, mobileservices 470→210, uiservice 271→255.
- Per-package AGENTS.md: added "Companion files" bullet; services gained a Mermaid architecture diagram in §2 and a Tests "what lives where" table in §3; gamedata §6 acknowledges PlayMode tests alongside EditMode.
- Host `README.md`: removed the absorbed `com.gamelovers.assetsimporter` row, added rows for `services`, `gamedata`, `mobileservices` with README links.
- Host `AGENTS.md`: fixed duplicate `## 4.` heading bug (now numbered `1.`–`8.` sequentially), expanded §6 to list all four packages, added §7 Documentation policy, added §8 Claude Code convention with full `CLAUDE.md` template.

### Data Flows
- Claude Code discovers a package's contributor guide via `CLAUDE.md` → `@AGENTS.md` native import → actual rules live in `AGENTS.md` only; Cursor and Codex continue to read `AGENTS.md` directly — single source, three tools.

## 2026-04-22 — Fix failing tests and editor-only test fixtures

### Architectural Decisions
- Introduced a shared runtime-compatible test-helpers assembly (`Tests/Helpers/` asmdef gated by `UNITY_INCLUDE_TESTS`) in `com.gamelovers.uiservice`, enabling EditMode and PlayMode tests to share MonoBehaviour-derived fixtures that `AddComponent<T>()` can legally attach to GameObjects.
- Preferred real pool fixtures over NSubstitute mocks for tests targeting self-referentially-constrained generic interfaces (e.g., `IObjectPool<T>` where `T : IPoolEntityObject<T>`) to work around a Castle DynamicProxy `ArgumentNullException: localType` IL-emission bug on Unity's Mono runtime.

### Structural Changes
- `com.gamelovers.services/Runtime/Pooling/ObjectPool.cs`: fixed iterate-while-mutating bug in `Despawn(bool onlyFirst, Func<T,bool>)` — forward index was not compensated after `SpawnedEntities.Remove`, so adjacent matches were skipped.
- `com.gamelovers.services/Tests/EditMode/Unit/ObjectPoolTest.cs`: rewrote `EntityDespawn_Successfully` to use a real `ObjectPool<IMockEntity>` instead of `Substitute.For<IObjectPool<IMockEntity>>()`.
- `com.gamelovers.services/Tests/EditMode/Unit/AssetResolverServiceTest.cs`: qualified `Object.DestroyImmediate(so)` as `UnityEngine.Object.DestroyImmediate(so)` to resolve CS0104 ambiguity introduced by `using System;` + `using UnityEngine;` in the same file.
- `com.gamelovers.gamedata/Tests/Editor/Unit/TestDataBuilders.cs` + `ConfigBrowser/ConfigValidationServiceTests.cs`: added `MockValidatableConfigAlt` sibling type; test now registers singleton and collection under two different types because `ConfigsProvider` keys its storage by `Type` and each type can hold only one role.
- `com.gamelovers.uiservice/Tests/Helpers/` (new): runtime-compatible helpers asmdef with `TestPresenter.cs` moved out of `Tests/EditMode/Helpers/`; both EditMode and PlayMode test asmdefs now reference it.
